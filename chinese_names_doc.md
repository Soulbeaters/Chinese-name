# Документация системы обработки китайских имен

## 1. Особенности китайских имен

### 1.1 Структурные особенности
- Китайские имена обычно состоят из фамилии и имени
- Фамилия ставится перед именем (в отличие от западных имен)
- Фамилия обычно состоит из одного иероглифа (например, Ли, Ван, Чжан), но встречаются и двусложные фамилии (например, Оуян, Сыма, Чжугэ)
- Имя обычно состоит из 1-2 иероглифов

### 1.2 Отсутствие понятия отчества
- В традиционных китайских именах нет понятия "отчества", как в западных именах
- В современных китайских именах могут быть сложные фамилии или сложные имена, но структура остается двоичной: "фамилия + имя"
- При обработке данных мы устанавливаем поле "отчество" китайского имени как пустую строку

### 1.3 Системы транслитерации
Основные системы транслитерации китайских имен включают:
- **Пиньинь**: наиболее широко используемая система транслитерации китайского языка для английского и других западных языков
  - Пример: 李明 → Li Ming
- **Система Палладия**: система транслитерации китайского языка для русского языка
  - Пример: 李明 → Ли Мин

### 1.4 Региональные различия
Китайские имена могут различаться в зависимости от региона:
- **Материковый Китай**: упрощенные иероглифы, обычно используется пиньинь
- **Тайвань**: традиционные иероглифы, может использоваться система Уэйда-Джайлза или пиньинь
- **Гонконг, Макао**: традиционные иероглифы, может использоваться кантонская транслитерация или другие системы
- **Зарубежные китайцы**: могут использовать различные системы транслитерации в зависимости от страны проживания

## 2. Алгоритмы обработки китайских имен

### 2.1 Распознавание китайских иероглифов
- Используются диапазоны Unicode для распознавания китайских иероглифов:
  - Базовые иероглифы: U+4E00-U+9FFF
  - Расширение A: U+3400-U+4DBF
  - Совместимые идеограммы: U+F900-U+FAFF

### 2.2 Алгоритм разделения имен
1. **Чистые китайские имена**:
   - Сначала проверяется, является ли фамилия двусложной (по предопределенному списку двусложных фамилий)
   - Если фамилия двусложная, первые 2-3 иероглифа считаются фамилией, остальные - именем
   - Если фамилия односложная, первый иероглиф считается фамилией, остальные - именем

2. **Транслитерированные китайские имена**:
   - Проверяется, присутствует ли фамилия в предопределенном списке транслитерированных фамилий
   - Определяется порядок имен (западный или китайский)
   - На основе положения фамилии определяется имя

### 2.3 Обработка различных форматов
Система может обрабатывать следующие форматы китайских имен:
- Чистый китайский формат: 李明, 王小红
- Пиньинь (китайский порядок): Li Ming, Wang Xiaohong
- Пиньинь (западный порядок): Ming Li, Xiaohong Wang
- Система Палладия (русская транслитерация): Ли Мин, Ван Сяохун
- Китайские имена в смешанных списках авторов

## 3. База данных китайских фамилий

### 3.1 Структура базы данных
Мы создали базу данных распространенных китайских фамилий, включающую:
- Односложные фамилии (например, Ли, Ван, Чжан)
- Двусложные фамилии (например, Оуян, Сыма, Чжугэ)
- Транслитерацию каждой фамилии на пиньинь
- Транслитерацию каждой фамилии по системе Палладия

### 3.2 Использование базы данных
- Для распознавания китайских фамилий
- Для распознавания транслитерированных фамилий и их соответствия китайским фамилиям
- Для правильного разделения китайских имен

### 3.3 Расширение базы данных
База данных поддерживает импорт и экспорт в формате JSON, что позволяет:
- Добавлять более редкие фамилии
- Добавлять региональные специфические фамилии
- Добавлять соответствия для других систем транслитерации

CHINESE_CHAR_RE = re.compile(r'[\u4e00-\u9fff\u3400-\u4dbf\uf900-\ufaff]')

### 4.2 Основные функции
Основные функции обработки включают:
- `is_chinese_char(char)`: определяет, является ли символ китайским иероглифом
- `is_chinese_name(name)`: определяет, является ли строка китайским именем
- `split_chinese_name(name)`: разделяет китайское имя на фамилию и имя
- `identify_transliterated_chinese_name(name)`: распознает транслитерированное китайское имя

### 4.3 Интеграция в существующую систему
Код интегрирован в существующую функцию `split_authors()`, процесс обработки следующий:
1. Сначала пытаемся распознать чистые китайские имена
2. Затем пытаемся распознать транслитерированные китайские имена
3. Если это не китайское имя, используем существующую логику обработки
4. В смешанных списках авторов одновременно обрабатываем разные типы имен

## 5. Часто задаваемые вопросы

### 5.1 Проблема "инициалов" в китайских именах
- В китайских именах обычно не используются сокращения или инициалы
- Сами китайские иероглифы являются самостоятельными семантическими единицами и не могут быть дальше сокращены
- При обработке мы не рассматриваем сокращенные формы китайских иероглифов

### 5.2 Обработка трехсложных и более длинных китайских имен
- Для трехсложных имен (например, 王小明) обычно первый иероглиф - фамилия, остальные два - имя
- Для четырехсложных имен это может быть двусложная фамилия + двусложное имя (например, 欧阳修齐) или односложная фамилия + трехсложное имя (встречается реже)
- Мы сначала проверяем наличие двусложной фамилии, затем по умолчанию считаем первый иероглиф фамилией

### 5.3 Совместимость различных систем транслитерации
- Система поддерживает две основные системы транслитерации: пиньинь и систему Палладия
- Расширение базы данных фамилий позволяет поддерживать другие системы транслитерации
- Система может распознавать разные порядки имен (китайский или западный) в транслитерированных именах

## 6. Примеры использования

### 6.1 Чистые китайские имена
```python
>>> split_authors("李明")
[('李', '明', '')]
>>> split_authors("欧阳修")
[('欧阳', '修', '')]
>>> split_authors("李明, 王小红")
[('李', '明', ''), ('王', '小红', '')]
```

### 6.2 Транслитерированные китайские имена
```python
>>> split_authors("Li Ming")
[('Li', 'Ming', '')]
>>> split_authors("Ming Li")  # западный порядок
[('Li', 'Ming', '')]
>>> split_authors("Ли Мин")  # система Палладия
[('Ли', 'Мин', '')]
```

### 6.3 Смешанные списки авторов
```python
>>> split_authors("李明, John Smith")
[('李', '明', ''), ('Smith', 'John', '')]
>>> split_authors("Wang Xiaohong, Ivan Petrov")
[('Wang', 'Xiaohong', ''), ('Petrov', 'Ivan', '')]
```
